Use this section to explore WebsiteTravel APIs by product area.

Each API should have its own reference section so authentication, base URLs, schemas, and endpoints stay grouped together.

Expect each API section to include:

- authentication requirements
- endpoint reference
- request and response schemas
- webhook events where available

If you are integrating with multiple products, start with the API section that matches the workflow you are building against.

Current extracted specs in this repo include the Agent / Reseller API, the Freelance Agent API, the EBS Supplier API, the Freelance Internal API, the Website Travel Supplier API, the Tourbuilder API, the Ibis API, the AccomSOA API, and the TransportSOA API.

<Tip>
  This repo is set up to use one OpenAPI specification per API. Add each spec to
  the `openapi/` folder and wire it into `docs.json`.
</Tip>
